## Directory Roles

- `reviews/`: dated, evidence-backed assessments. A review describes what was
  observed; it is not an implementation status ledger.
- `architecture/`: approved or proposed system blueprints and durable design
  boundaries.
- `decisions/`: accepted durable architecture decisions (ADRs).
- `product/`: product intent, UX policy, quality bars, and non-goals.
- `ai/`: current-state and workflow documents for future coding agents.
- `../progress/`: active PLAN+TASK documents and historical closeouts.
- `../doc/`: public end-user guides and legacy product evaluation material.

Historical files are retained for traceability. Current claims must come from
`ai/current-reality.md` and the active PLAN+TASK document, not from old
closeouts or dated evaluation snapshots.
