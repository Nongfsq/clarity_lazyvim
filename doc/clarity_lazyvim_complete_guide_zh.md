# Clarity LazyVim 中文完全指南

适用人群：

- 用过 VS Code、记事本、Typora，但不熟悉 Neovim
- 知道一点 Vim，但很多命令已经忘了
- 在 Windows 11 + WSL2 里工作
- 想把这套配置当成日常工具，而不是一次性折腾玩具

这份文档的目标不是“把 Vim 讲成教材”，而是让你在真实工作里顺手用起来。

## 1. 先记住 10 个最有用的动作

如果你现在就要开始用，只记住这一页就够了。

| 目标 | 按键 / 命令 | 说明 |
| --- | --- | --- |
| 看帮助菜单 | `空格` 然后停一下 | `which-key` 会弹出菜单 |
| 搜文件 | `空格 f f` | 相当于 VS Code 的 `Ctrl + P` |
| 搜文本 | `空格 f w` | 相当于 VS Code 的 `Ctrl + Shift + F` |
| 打开文件树 | `空格 e` | 左侧项目树 |
| 开始输入 | `i` | 进入插入模式 |
| 退出输入 | `Esc` | 回到普通模式 |
| 保存 | `:w` | 先按 `Esc` 再输 |
| 保存并退出 | `:wq` | 最常用退出方式 |
| 跳到定义 | `gd` | 看函数、变量定义 |
| 看当前报错 | `gl` | 查看当前行诊断信息 |

你只要会这 10 个动作，就已经能完成大部分日常工作。

如果你只额外记一个“救命入口”，请记住：

```vim
:ClarityStart
```

或者：

```text
空格 h h
```

当前版本在“第一次空启动”时，会自动弹出这个引导浮窗一次。

如果你之后想再看，随时可以重新打开：

- `:ClarityStart`
- `空格 h h`

## 2. 这个项目到底是什么

`clarity_lazyvim` 不是“一个全新的编辑器软件”，它是：

- `Neovim` 这个编辑器引擎
- `LazyVim` 这个成熟底座
- 再加上本仓库自己的高对比主题、终端工作流、Git 工作流、审计命令和一部分更适合中文用户记忆的快捷键

你可以这样理解：

- `Neovim` 负责编辑
- `LazyVim` 负责生态和默认能力
- `clarity_lazyvim` 负责把它变成更适合长期使用的个人/团队工作台

这个项目当前最值得保留的东西有三个：

1. 可读性很强的主题
2. 比纯手搓配置更稳定的 LazyVim 底座
3. 已经开始成型的“最小必要插件集”

### 2.1 这个项目现在支持中文和英文界面

当前版本已经把“你看到的界面文字”和“源码里的维护性注释”分开处理了。

你在编辑器里会直接接触到的 Clarity 自有界面，现在支持：

- 中文
- 英文
- 自动检测

当前已经支持切换的主要范围有：

- `ClarityStart`、`ClarityClipboard`、`ClaritySync` 这些帮助面板
- 按 `空格` 弹出的主菜单提示
- Clarity 自己定义的快捷键说明
- Clarity 自己定义的命令说明
- Clarity 自己弹出的提示消息

### 2.2 为什么源码注释还是英文

因为这两类东西服务的对象不同：

- 运行时界面是给用户看的，所以应该可以本地化
- 源码注释是给维护者和未来协作者看的，所以应该统一成英文

这样做的好处是：

- 你自己用起来仍然可以是中文界面
- 代码本身不会变成中英混杂、难以维护的状态

### 2.3 怎么切换界面语言

在 Neovim 里输入：

```vim
:ClarityLanguage
```

它会告诉你当前语言状态。

如果你想切换成中文：

```vim
:ClarityLanguage zh
```

如果你想切换成英文：

```vim
:ClarityLanguage en
```

如果你想交给系统自动判断：

```vim
:ClarityLanguage auto
```

这个选择会自动保存，下次打开 Neovim 还会继续生效。

但有一点你要记住：

- 切换后，部分键位说明和命令菜单需要重启 Neovim 才会完整刷新

所以最稳的做法是：

1. 执行 `:ClarityLanguage zh` 或 `:ClarityLanguage en`
2. 退出 Neovim
3. 重新打开

另外要注意：

- 这次切换的是 Clarity 自己拥有的界面
- 某些上游插件自己的英文界面，不一定会一起变成中文

## 3. 你现在身处哪一层

很多混乱都不是“你不会用”，而是你把不同层混在了一起。

你现在通常会同时处在 4 层：

1. Windows 11
2. Windows Terminal
3. WSL Ubuntu
4. Neovim / LazyVim / clarity_lazyvim

这 4 层不是一回事。

### 3.1 Windows 11

你平常的浏览器、微信、文件资源管理器都在这里。

### 3.2 Windows Terminal

这是黑底终端窗口，它负责显示命令和接收键盘输入。

### 3.3 WSL Ubuntu

这是 Windows 里的 Linux 环境。

你看到：

- `/home/frank`
- `~`
- `bash`
- `zsh`

这些都是 Linux 层。

### 3.4 Neovim

当你在终端里输入：

```bash
nvim
```

你才真正进入编辑器。

所以这三件事是不同的：

- `Ctrl + Shift + C` 是终端复制
- `yy` / `p` 是 Neovim 内部复制粘贴
- `:w` / `:q` 是 Neovim 命令

把层分清楚，后面 80% 的困惑都会少很多。

## 5. 先掌握模式，不要先背一百个命令

Neovim 最重要的不是命令数量，而是“模式”。

### 5.1 普通模式

这是默认模式。

在普通模式下，按键主要表示动作，不是直接输入文字。

例如：

- `j` 下移
- `dd` 删一行
- `yy` 复制一行

### 5.2 插入模式

这时才像普通编辑器一样输入文字。

进入方式：

```text
i
```

### 5.3 可视模式

这时用于选中文本。

进入方式：

```text
v
```

### 5.4 你最安全的习惯

只要你有一点不确定，就先按：

```text
Esc
```

这是“回到可控状态”的最快方式。

## 6. 忘记命令时，不要硬想

这套配置最推荐的“防遗忘机制”有两个。

### 6.1 按 `空格`，停一下

`<leader>` 默认就是空格键。

所以你按一下：

```text
空格
```

然后停半秒，`which-key` 会显示当前能走的命令分组。

这是你最应该养成的习惯。

不要逼自己把所有命令背下来。

### 6.2 搜快捷键

如果你只记得“好像有个快捷键能做这事”，直接用：

```text
空格 s k
```

它会打开当前可用快捷键列表，你可以搜索。

这是“忘了命令怎么办”的官方解决路径。

### 6.3 `ClarityStart` / `空格 h h`（产品内恢复入口）

当前版本已经提供一个统一入口：

- `:ClarityStart`
- `空格 h h`

并且在第一次空启动时会自动弹出一次，帮助你快速进入状态。

它的目标不是替代文档，而是当你忘了命令时，直接在编辑器里给你一条可执行路径，集中显示：

- 主路径命令
- 剪贴板说明
- 环境审计与验证入口

如果你当前版本没有这个入口，通常说明你跑的还是旧配置。临时替代路径是：

- `空格` 看 which-key 分组
- `空格 s k` 搜索快捷键

### 6.4 另外四个你应该知道的帮助命令

如果你在 Windows 11 + WSL2 中长期使用，这 4 个命令非常值得记住：

- `:ClarityClipboard`
  专门解释复制、粘贴、系统剪贴板、终端复制之间的区别
- `:ClaritySync`
  专门解释 Windows 主仓、WSL 运行仓、为什么“改了却没生效”
- `:ClarityValidate`
  专门检查关键键位和特殊 UI 行为是不是仍然正常
- `:ClarityLanguage`
  专门查看或切换 Clarity 自己的界面语言

## 11. 复制、粘贴、选择

这是 WSL 用户最容易迷糊的一组。

### 11.1 编辑器内部复制

- 复制一行：`yy`
- 粘贴到后面：`p`
- 粘贴到前面：`P`

### 11.2 选择一段文字

```text
v
```

然后移动光标选择，最后：

- 复制：`y`
- 删除：`d`

### 11.3 复制到 Windows 剪贴板

当前配置已经开启 `clipboard=unnamedplus`。

这意味着：

- 在 clipboard provider 正常时，普通 `y` / `yy` 往往已经会进入系统剪贴板
- 如果你想显式指定系统剪贴板，仍然可以使用 `+` 寄存器

最明确的写法是：

```vim
"+y
```

复制当前行：

```vim
"+yy
```

复制整个文件：

```vim
:%y+
```

### 11.4 从 Windows 粘贴回来

在 Windows Terminal 里最常见的是：

```text
Ctrl + Shift + V
```

不是所有终端都默认用 `Ctrl + V`。

### 11.5 为什么 `Ctrl + Shift + C` 有时没用

因为：

- 它复制的是终端选中的文字
- 不是 Neovim 内部的可视选择

所以你要分清：

- 终端复制：鼠标选中后 `Ctrl + Shift + C`
- Neovim 复制到系统：`"+y`

## 16. Git：现在建议分成两层来记

你可以先把 Git 能力分成两种：

1. 看改动
2. 操作改动

### 16.1 全局 Git 浏览仍然走 `g` 组

```text
空格 g d
```

这是“看仓库级 Git 信息”的路径。

你可以先记这些：

- `空格 g d`
  看 Git diff 列表
- `空格 g s`
  看 Git status
- `空格 g b`
  看当前行相关 Git 历史

### 16.2 Hunk 操作现在走 `h` 组

如果你想操作“当前文件里的这一块改动”，请记 `h` 组：

### 16.2.1 预览当前 hunk

```text
空格 h p
```

### 16.2.2 暂存当前 hunk

```text
空格 h s
```

### 16.2.3 重置当前 hunk

```text
空格 h r
```

### 16.2.4 看当前行 blame

```text
空格 h b
```

### 16.2.5 看当前文件 diff

```text
空格 h d
```

### 16.2.6 跳到上一个 / 下一个 hunk

- 上一个：`[h`
- 下一个：`]h`

### 16.3 为什么现在这样更好

因为现在你可以这样理解：

- `g` = Git 仓库级浏览
- `h` = 当前文件改动块操作

这比以前全部挤在 `g` 组更容易记。

## 17. 审计、健康检查、依赖

### 17.1 最有用的体检命令

```vim
:ClarityAudit
```

它会检查：

- 仓库布局是否正确
- 必需工具是否存在
- 可选工具是否存在
- 插件启用状态
- 整体环境 readiness

### 17.2 终端里跑审计

```powershell
python scripts/run_clarity_audit.py
```

### 17.3 通用健康检查

```vim
:checkhealth
```

### 17.4 什么时候先跑审计

出现以下情况时，先跑 `:ClarityAudit`：

- 搜索突然不可用
- Copilot 启动失败
- 格式化失败
- 新机器刚配置完
- WSL / Windows 切换之后感觉环境不对

### 17.5 进阶验证（T-011 方向）

除 `:ClarityAudit` 之外，当前还可以直接运行：

```vim
:ClarityValidate
```

或在终端执行：

```powershell
python scripts/run_clarity_validate.py
```

它会覆盖“行为正确性”，不仅是“工具是否安装”。

当前验证覆盖：

- Windows + Ubuntu (WSL) 启动 smoke test
- 核心命令断言：
  - `空格 f f`
  - `空格 f w`
  - `空格 g d`
  - `空格 h s`
- 特殊界面行为检查：
  - dashboard
  - neo-tree
  - terminal
- provider 就绪度检查：
  - clipboard
  - `pynvim`
  - `npm neovim`
  - Copilot Node 版本
- 本地化检查：
  - `en` / `zh` 键位说明是否正确
  - `:ClarityLanguage` 命令是否存在
  - 中英文翻译键是否齐全

### 17.6 你怎么看验证结果

你可以这样理解：

- `:ClarityAudit`
  更像“环境体检”，看依赖、provider、插件状态、整体 readiness
- `:ClarityValidate`
  更像“行为验收”，看命令和界面有没有回归

当前项目的真实状态是：

- Windows 主开发环境：`94/100`
- WSL 运行环境：`100/100`

这不是说 Windows 不能用，而是说当前 Windows 上还缺少两个可选项：

- `fd`
- `htop` / `btop`

它们不会阻止你工作，只会让某些体验没有达到“最完整状态”。

## 20. Windows 11 + WSL2 用户最常见问题

### 20.1 为什么复制不到 Windows

因为你把“终端复制”和“Neovim 内部复制”混在一起了。

最稳方案：

- 在 provider 正常时直接用 `y` / `yy`
- 想强制走系统剪贴板时用 `"+y`
- 从系统粘贴：`Ctrl + Shift + V`

### 20.2 为什么我按 `空格 f f` 或 `空格 f w` 报错

如果你看到的是 `Telescope ...` 报错，那说明你当前跑的还是旧配置。

本仓库当前正确搜索后端已经改成了 `Snacks picker`。

解决方式通常是：

1. 同步到最新仓库
2. 重启 Neovim
3. 重新执行命令

### 20.3 为什么左侧行号不像普通行号

当前普通编辑窗口已经强制使用绝对行号。

但像文件树、终端、特殊窗口会使用不同显示逻辑，这是正常的。

### 20.4 为什么有些命令有时能用有时不能

因为有些能力依赖外部工具：

- Copilot 依赖 Node.js `22+`
- 全局搜索依赖 `rg`
- 文件搜索更推荐 `fd`
- 系统监视终端依赖 `htop` / `btop`

### 20.5 为什么我“已经改了”，WSL 里还是旧行为

这通常不是你操作错，而是你在两份仓库之间没有同步到同一个提交。

官方建议流程：

1. 在 Windows 主仓改动并 `commit`、`push`
2. 进入 WSL 运行仓 `pull`
3. 重启 Neovim 再验证命令

把这条流程记成一句话：

`Windows 写入 -> 远端同步 -> WSL 拉取 -> Neovim 重启`

### 20.6 为什么我切换成中文或英文后，有些地方像是没完全变

这是因为有一部分说明文字是在启动时注册的。

所以 `:ClarityLanguage` 做了两件事：

1. 立即保存你的语言选择
2. 提醒你重启 Neovim，以便完整刷新键位说明和命令菜单

最稳操作：

1. 执行 `:ClarityLanguage zh` 或 `:ClarityLanguage en`
2. `:q` 退出
3. 重新 `nvim`

另外也要知道：

- Clarity 自己的帮助面板、提示和说明会跟着切换
- 上游插件自己的界面，不一定都支持中文
